# 🕐 Linux Kernel Device Driver Clock


리눅스 커널 공간(Kernel Space)에서 직접 하드웨어를 제어하는 **통합 디바이스 드라이버 기반 임베디드 시계 시스템**입니다.  
별도의 유저 공간(User Space) 프로그램 없이, 커널 드라이버 자체적으로 시간 설정 및 UI 제어가 가능합니다.

---

## 1. 개요

DS1302 RTC 모듈을 통해 정확한 시간을 유지하고, I2C OLED 디스플레이에 현재 시간을 시각화하며,  
별도의 유저 공간 프로그램 없이 커널 드라이버 자체적으로 시간 설정 및 UI 제어 가능한 임베디드 시계 시스템 구축


---

## 2. 구성도 (System Architecture)


![하드웨어](https://github.com/user-attachments/assets/f4e531e2-0111-4c88-8404-bafc1c5e26d8)

![회로도](https://github.com/user-attachments/assets/c8581803-6e0e-436b-8780-ae711ea619ab)


- **Raspberry Pi 4 :** 메인 제어 플랫폼, 커널 드라이버 실행 환경
- **DS1302 (RTC 모듈) :** GPIO 기반 3-wire 프로토콜로 정확한 시간 유지. 전원이 꺼져도 배터리로 시간 보존
- **SSD1306 (OLED 모듈) :** I2C 프로토콜(Address: `0x3C`)로 연결, 현재 시간 및 UI 시각화
- **EC11 (로터리 엔코더) :** 인터럽트 기반으로 회전/버튼 입력 처리, FSM 상태 전환 트리거

---

## 3. 주요 기능

1. **FSM(Finite State Machine) 기반 모드 제어**  
   선택 모드와 수정 모드 간의 상태 전환 구현

2. **로터리 엔코더 입력**  
   모드간 회전으로 값 증가/감소 또는 포커스 이동 및 클릭으로 확인 구현

3. **커널 레벨 하드웨어 제어**  
   별도 응용 프로그램 없이 커널 모듈 내에서 GPIO 및 I2C 프로토콜을 제어

4. **비동기 화면 갱신**  
   인터럽트 발생 시 I2C 통신을 Workqueue를 통해 시스템 안정성 확보

5. **RTC(DS1302) 동기화**  
   전원이 꺼져도 시간이 유지되며, 변경된 시간을 즉시 하드웨어에 반영




## 4. 요소 기술
###FSM (Finite State Machine)

![FSM](https://github.com/user-attachments/assets/20b6b8af-51f1-46fd-af2a-b288a75a6516)


### Workqueue
- 리눅스 커널의 **지연 처리 기법**
- 인터럽트 문맥에서 처리하기 어려운 긴 작업이나 대기가 필요한 작업을 프로세스 문맥으로 미루어 처리
- 커널 스레드 환경에서 실행되므로 I2C 통신 등 대기 함수(`sleep`) 사용 가능

### I2C (Inter-Integrated Circuit)
- SDA, SCL 두 개의 선만 사용하는 직렬 통신 프로토콜
- Master-Slave 구조: 마스터가 통신을 주도하고 슬레이브가 응답
- 각 장치는 고유 주소를 가지므로 하나의 버스에 여러 장치 연결 가능
- 리눅스 커널의 I2C 서브시스템(`i2c_master_send`, `i2c_master_recv`) 활용

### Interrupt
- CPU가 실행 중 예기치 않은 이벤트 발생 시 현재 작업을 중단하고 즉시 처리하는 메커니즘
- 로터리 엔코더 회전 및 버튼 입력 처리
- 타이머 인터럽트를 이용한 주기적 화면 동기화
- **제약사항**: ISR 내에서는 처리가 빠르게 끝나야 하며, `sleep` 함수 사용 불가

## 5. 트러블 슈팅 (Troubleshooting)

### 1. 커널 모듈 로드 시 라즈베리 파이 꺼짐

| | 내용 |
|--|------|
| **현상** | 커널 모듈을 로드하면 라즈베리 파이가 꺼짐 |
| **원인** | 인터럽트 핸들러 내에서 `sleep` 함수를 호출하여 커널 패닉 발생 |
| **해결** | Workqueue로 `sleep`이 필요한 작업을 프로세스 문맥으로 이동 |

### 2. OLED 화면 노이즈

| | 내용 |
|--|------|
| **현상** | OLED 화면에 노이즈가 발생하여 정상적으로 출력되지 않음 |
| **원인** | 다른 테스트 드라이버가 이미 I2C 드라이버 자원을 점유하여 OLED 드라이버 초기화 실패 |
| **해결** | 시스템 재부팅을 통해 자원 점유 상태 초기화 |

### 3. RTC 비정상적인 날짜/시간 출력

| | 내용 |
|--|------|
| **현상** | `165-165-165 165:165:165` 등 비정상적인 날짜와 시간이 지속 출력 |
| **원인** | 하드웨어 배선 불량 추정 |
| **해결** | RTC 점퍼 와이어를 새로 교체하여 물리적 접촉 불량 문제 해결 |

## 6. 개발 후기

- **Workqueue는 MCU의 `main loop`와 유사한 역할**: 커널에서 지속적으로 처리해야 할 작업을 안전하게 위임하는 구조
- 인터럽트 핸들러 내에서 `sleep` 사용 불가 등 **커널 레벨 제약사항**을 몸소 체험
- 커널 Rust API의 한계 체감: GPIO, Timer 관련 API가 아직 완전하지 않음
- C 커널과 다른 관점의 API를 통해 Rust 커널 개발자들의 철학과 불만 사항을 이해
